# Chapter 6: Temporal-Difference Learning

## Core Idea

Temporal-difference methods replace the full-return Monte Carlo target with a one-step bootstrapped target. Instead of waiting for the episode to end, they update immediately after each transition.

## TD(0) Prediction

For policy evaluation, the TD(0) target is

`R_{t+1} + gamma V(S_{t+1})`

and the update is

`V(S_t) <- V(S_t) + alpha [R_{t+1} + gamma V(S_{t+1}) - V(S_t)]`

[td_eval.py](/Users/HyeongjinKim/OneDrive/Desktop/EmbodiedRL_Lab/deep-learning-from-scratch-4/ch06/td_eval.py) implements exactly this rule.

The bracketed term is the TD error:

`delta_t = R_{t+1} + gamma V(S_{t+1}) - V(S_t)`

This `delta` becomes one of the most important objects in RL.

## Why TD Is Different from Monte Carlo

- Monte Carlo target: unbiased but high variance.
- TD target: biased because it uses the current estimate `V(S_{t+1})`, but lower variance and available immediately.

This is a classic bias-variance tradeoff.

## SARSA

SARSA is an on-policy control algorithm. Its target is

`R_{t+1} + gamma Q(S_{t+1}, A_{t+1})`

The name comes from the tuple

`(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})`

[sarsa.py](/Users/HyeongjinKim/OneDrive/Desktop/EmbodiedRL_Lab/deep-learning-from-scratch-4/ch06/sarsa.py) keeps a deque of length 2 because the update for the previous transition must wait until the next action `A_{t+1}` is actually sampled.

Because SARSA follows the same policy it learns about, it tends to reflect the consequences of exploration.

## Q-Learning

Q-learning is off-policy. Its target is

`R_{t+1} + gamma max_a Q(S_{t+1}, a)`

This corresponds to the Bellman optimality equation rather than the value of the behavior policy. [q_learning.py](/Users/HyeongjinKim/OneDrive/Desktop/EmbodiedRL_Lab/deep-learning-from-scratch-4/ch06/q_learning.py) therefore maintains:

- a greedy target policy `pi`,
- an exploratory behavior policy `b`.

The behavior policy collects data; the target policy defines the learning objective.

## On-Policy vs Off-Policy

On-policy:

- learn the value of the policy actually used to generate data,
- example: SARSA.

Off-policy:

- learn about one policy using data generated by another,
- example: Q-learning and off-policy SARSA.

This distinction becomes central later in DQN and actor-critic variants.

## Off-Policy SARSA and Importance Sampling

[sarsa_off_policy.py](/Users/HyeongjinKim/OneDrive/Desktop/EmbodiedRL_Lab/deep-learning-from-scratch-4/ch06/sarsa_off_policy.py) applies an importance ratio

`rho = pi(A_{t+1} | S_{t+1}) / b(A_{t+1} | S_{t+1})`

to correct the one-step target. This is the TD analogue of the importance-sampling idea introduced in chapter 5.

## Bootstrapping

Bootstrapping means that an estimate is updated toward a target containing another learned estimate. In TD methods:

- `V(S_t)` is updated using `V(S_{t+1})`,
- `Q(S_t, A_t)` is updated using `Q(S_{t+1}, A_{t+1})` or `max_a Q(S_{t+1}, a)`.

Bootstrapping is why TD methods can learn before an episode ends, but it is also why they can propagate approximation errors.

## Implementation Insights

- The terminal case always sets the bootstrap component to `0`. This encodes the fact that there is no discounted future after termination.
- In SARSA, the extra final dummy update is an implementation trick to let the last real transition receive a terminal target.
- `greedy_probs` turns learned Q-values into epsilon-greedy policies, making policy improvement an implicit part of every update.
- Q-learning is often more optimistic than SARSA because it evaluates the best next action, not the actually sampled exploratory one.

## Connection to Function Approximation

Tabular TD assumes values can be stored explicitly for every state or state-action pair. Once the state space becomes large, we replace tables by parameterized functions. That is chapter 7.
