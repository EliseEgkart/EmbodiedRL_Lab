# Chapter 2: 마코프 과정과 MDP 기초

## 저장소 상태

이 저장소에는 `ch02` 실행 코드가 없어서, 이 문서는 밴딧에서 동적계획법으로 넘어가기 전에 필요한 수학을 메우는 역할을 합니다.

## 왜 상태가 필요한가

밴딧은 한 번 행동하고 바로 보상만 받는 구조입니다. 하지만 일반 강화학습에서는 행동이 미래 상태를 바꾸고, 그 미래 상태가 다시 장기 보상에 영향을 줍니다. 이 구조를 다루려면 상태 기반 확률 과정이 필요합니다.

## 마코프 성질

마코프 성질은 미래가 현재 상태만 알면 충분히 결정된다는 가정입니다.

`P(S_{t+1} | S_t, S_{t-1}, ..., S_0) = P(S_{t+1} | S_t)`

이 가정이 없으면 상태가 충분통계량이 아니어서 가치함수를 상태 기준으로 정의하기 어렵습니다.

## 마코프 보상 과정(MRP)

MRP는 다음으로 구성됩니다.

- 상태공간 `S`
- 전이확률 `P`
- 보상함수 `R`
- 할인율 `gamma`

반환값(return)은

`G_t = R_{t+1} + gamma R_{t+2} + gamma^2 R_{t+3} + ...`

이고, 상태가치는

`V(s) = E[G_t | S_t = s]`

입니다.

## 할인율의 의미

`gamma`는 단순 하이퍼파라미터가 아니라 다음 의미를 동시에 갖습니다.

- 먼 미래 보상의 현재 중요도 조절
- 무한합 수렴 보장
- Bellman 연산자의 수축성 보장

그래서 `gamma`가 커질수록 장기적인 영향은 더 많이 반영되지만, 학습 분산과 수렴 난이도도 커질 수 있습니다.

## 마코프 결정 과정(MDP)

MDP는 MRP에 행동을 추가한 것입니다.

`p(s', r | s, a)`

는 상태 `s`에서 행동 `a`를 했을 때 다음 상태 `s'`와 보상 `r`이 어떻게 나오는지를 나타내는 환경 모델입니다.

## 정책

정책은 상태 조건부 행동분포입니다.

`pi(a | s) = P(A_t = a | S_t = s)`

결정적 정책은 한 행동만 선택하고, 확률적 정책은 여러 행동에 확률을 분배합니다. epsilon-greedy도 확률적 정책입니다.

## 상태가치와 행동가치

고정 정책 `pi`에 대해

`V^pi(s) = E_pi[G_t | S_t = s]`

`Q^pi(s, a) = E_pi[G_t | S_t = s, A_t = a]`

입니다.

- `V^pi(s)`: 이 상태 자체가 얼마나 좋은가
- `Q^pi(s, a)`: 이 상태에서 특정 행동이 얼마나 좋은가

이후 코드 대부분은 결국 두 문제 중 하나를 풉니다.

## 구현 통찰

- `reset()`과 `step(action)` 인터페이스는 사실상 MDP 샘플러입니다.
- `next_state, reward, done`만으로 강화학습 업데이트를 짤 수 있는 이유는 마코프 가정 때문입니다.
- 상태가 충분통계량이 되면 값을 `state` 혹은 `(state, action)`에 붙여 저장할 수 있습니다.
