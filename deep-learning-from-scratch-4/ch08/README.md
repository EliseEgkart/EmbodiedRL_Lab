# Chapter 8: Deep Q-Networks

## Why Naive Neural Q-Learning Is Unstable

If we directly combine Q-learning with a neural network, three problems appear:

- consecutive samples are highly correlated,
- the target depends on the same parameters being updated,
- bootstrapping can amplify approximation errors.

DQN addresses these with replay memory and a target network.

## DQN Target

For a minibatch transition `(s, a, r, s', done)`, the DQN target is

`y = r + (1 - done) * gamma * max_{a'} Q_{theta^-}(s', a')`

where:

- `theta` are online-network parameters,
- `theta^-` are target-network parameters held fixed for several updates.

The loss is

`L(theta) = E[(Q_theta(s, a) - y)^2]`

## Experience Replay

Replay memory stores transitions

`(s, a, r, s', done)`

and samples random minibatches from the buffer.

Mathematical motivation:

- SGD works better when samples are closer to i.i.d.,
- random replay weakens temporal correlation,
- old experiences can be reused multiple times, improving sample efficiency.

## Target Network

If the same network produced both the current prediction and the target, then every parameter update would also move the target, creating a highly non-stationary regression problem.

The target network freezes parameters for a while:

- online network `Q_theta` is optimized every step,
- target network `Q_{theta^-}` is synchronized periodically.

This makes the target piecewise stationary and significantly stabilizes learning.

## Bellman Residual Minimization

DQN can be interpreted as approximate minimization of a Bellman residual over sampled transitions. But unlike supervised learning, the labels are not externally given; they are generated by the agent's own target network and reward stream.

That is the central difficulty of value-based deep RL.

## CartPole as a Control Problem

CartPole has:

- continuous state features,
- small discrete action space,
- episodic undiscounted-style reward of `+1` per alive step.

Because the state is continuous, tabular Q-learning is impossible without discretization. This is exactly the regime where function approximation becomes necessary.

## Epsilon-Greedy in Deep RL

Exploration is still handled by epsilon-greedy:

- random action with probability `epsilon`,
- greedy action from the network otherwise.

This is mathematically simple but often sufficient for small discrete-action environments.

## Semi-Gradient Detail

The target branch must be detached from gradient flow. Otherwise the optimizer would try to move both the prediction and the label simultaneously, which breaks the intended Bellman regression interpretation.

This is why the code explicitly stops gradients through `next_q`.

## Implementation Insights

- `np.stack` in the replay buffer is not a cosmetic detail; it turns a list of transitions into a minibatch tensor suitable for vectorized linear algebra.
- The target-network sync interval is a hyperparameter controlling the tradeoff between target freshness and target stability.
- Replay memory size controls how diverse the sampled training distribution can be.
- DQN is still fundamentally off-policy because it learns about the greedy policy while data is generated by an exploratory behavior policy.

## Limitations

Vanilla DQN can still suffer from:

- overestimation bias due to the max operator,
- sensitivity to reward scaling and optimizer settings,
- instability in harder environments.

Those issues motivate later extensions such as Double DQN, dueling networks, prioritized replay, and distributional RL.
