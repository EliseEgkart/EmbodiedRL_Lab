# Chapter 1: 밴딧 수학 정리

## 개요

`ch01`은 상태 전이가 없는 가장 단순한 강화학습 문제인 멀티 암드 밴딧을 다룹니다. 핵심은 MDP로 가기 전에 탐험과 활용의 충돌을 수학적으로 분리해서 보는 것입니다.

## 핵심 확률변수

- `A_t`: 시각 `t`에 선택한 행동
- `R_t`: 그 행동 이후 관측한 보상
- 각 팔 `a`에는 보이지 않는 진짜 기대보상 `q_*(a)`가 존재

베르누이 밴딧에서는

`q_*(a) = P(R_t = 1 | A_t = a)`

이며 보상은 보통 `0` 또는 `1`입니다.

## 행동가치 함수

진짜 행동가치는

`q_*(a) = E[R_t | A_t = a]`

이고, 이를 표본으로 추정하는 값이 `Q(a)`입니다.

정상(stationary) 환경이라면 `n`번 관측 후 표본평균은

`Q_n(a) = (1 / n) * sum_{i=1}^n R_i(a)`

입니다.

## 증분 업데이트

직접 평균을 다시 계산하지 않고도

`Q_n = Q_{n-1} + (R_n - Q_{n-1}) / n`

으로 같은 값을 얻을 수 있습니다. 여기서

`R_n - Q_{n-1}`

은 예측오차이며, 이후 TD 오차와 확률적 경사하강법의 원형이 됩니다.

[avg.py](./avg.py)와 [bandit.py](./bandit.py)의 핵심은 바로 이 식입니다.

## 탐험과 활용

현재 추정치만 믿고 항상 최댓값 행동만 선택하면 초기에 운 좋게 높게 나온 비최적 팔에 고착될 수 있습니다. 이를 막기 위해 epsilon-greedy를 사용합니다.

- 확률 `epsilon`으로 무작위 탐험
- 확률 `1 - epsilon`으로 현재 최선 행동 선택

행동 수가 `k`일 때 정책은

`pi(a_greedy) = 1 - epsilon + epsilon / k`

`pi(a_non_greedy) = epsilon / k`

가 됩니다.

## 누적 후회(regret)

이론적으로 자주 보는 양은 누적 후회입니다.

`Regret(T) = sum_{t=1}^T [q_*(a^*) - q_*(A_t)]`

코드에서 직접 계산하지는 않지만, [bandit_avg.py](./bandit_avg.py)의 평균 보상 곡선은 사실상 후회를 얼마나 줄이고 있는지 간접적으로 보여줍니다.

## 비정상 환경과 상수 보폭

환경이 변하지 않는다면 표본평균이 자연스럽습니다. 하지만 환경이 시간에 따라 바뀌면 오래된 데이터는 덜 믿어야 합니다. 이때

`Q <- Q + alpha * (R - Q)`

를 쓰면 최근 데이터에 더 큰 가중치가 들어갑니다.

이를 전개하면

`Q_t = (1 - alpha)^t Q_0 + sum_{i=1}^t alpha (1 - alpha)^{t-i} R_i`

가 되어, 지수이동평균 구조가 나타납니다. [non_stationary.py](./non_stationary.py)가 이 차이를 실험합니다.

## 구현 통찰

- `reward - Q` 형태의 오차는 이후 장 전체를 관통하는 핵심 패턴입니다.
- 비정상 문제에서 표본평균이 밀리는 이유는 “오래된 관측을 너무 공평하게 대우”하기 때문입니다.
- 여러 번 독립 실행 후 평균을 내는 이유는 밴딧 실험의 분산이 크기 때문입니다.
- Chapter 1은 추정기가 환경 가정과 맞아야 한다는 점을 가장 간단한 형태로 보여줍니다.
